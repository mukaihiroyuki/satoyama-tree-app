## 5. Unhandled Async Calls

3 Supabase `.from()` chains use `.then()` without a `.catch()` handler for promise rejection.

| File | Lines | Issue |
|------|-------|-------|
| `src/app/estimates/[id]/page.tsx` | 203–205 | `.then(({ data }) => …)` with no `.catch()` — network errors silently dropped |
| `src/app/shipments/[id]/picking/page.tsx` | 177–183 | `.then(({ error }) => …)` with no `.catch()` — unhandled rejection on network failure |
| `src/app/shipments/[id]/picking/page.tsx` | 186–192 | `.then(({ error }) => …)` with no `.catch()` — unhandled rejection on network failure |

**Details:**

### `src/app/estimates/[id]/page.tsx:203`
```typescript
supabase.from('clients').select('id, name, default_rate').order('name').then(({ data }) => {
    setClients(data || [])
})
```
No error destructuring and no `.catch()`. If the request fails, `clients` stays empty with no user feedback.

### `src/app/shipments/[id]/picking/page.tsx:177`
```typescript
supabase
    .from('shipment_items')
    .update({ picked_at: now })
    .eq('id', item.id)
    .then(({ error }) => {
        if (error) console.error('[picking] picked_at update failed:', error)
    })
```
Handles Supabase-level errors but `.then()` itself can reject on network failure — no `.catch()`.

### `src/app/shipments/[id]/picking/page.tsx:186`
```typescript
supabase
    .from('shipments')
    .update({ picking_status: 'in_progress' })
    .eq('id', shipment.id)
    .then(({ error }) => {
        if (error) console.error('[picking] status update failed:', error)
    })
```
Same pattern as above.

**Action:** Add `.catch((err) => console.error('...', err))` to all three chains, or rewrite as `async/await` with try/catch (preferred per project conventions).
