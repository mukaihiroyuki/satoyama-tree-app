### [5] Supabase エラーチェック漏れ（4件 open、1件 partial）

**5-1 🔶 Partial** — `src/lib/activity-log.ts` line 27: `logActivity` 内の `.insert()` は try/catch で囲まれたが、Supabase は DB エラーで throw しない（`{ data, error }` を返す）。`error` を destructure してチェックする必要がある。`logActivityBulk` は修正済み（`if (error) throw error` あり）。

```ts
// 現状（activity-log.ts:27）
await supabase.from('activity_logs').insert({ ... })
// 修正案
const { error } = await supabase.from('activity_logs').insert({ ... })
if (error) throw error
```

**5-2 / 5-3 / 5-4 ⚠️ Open** — `src/app/c/[clientId]/s/[shipmentId]/page.tsx` の3クエリ（line 51, 75, 101）は `{ data }` のみ destructure、`error` 未チェック。DBエラーが「データなし」と区別不能。

### [SEC-1] API ルート認証チェック（変化なし）

`src/app/api/label/[id]/route.ts` — `auth.getUser()` なし。tree ID を知れば未認証ユーザーが任意ラベルを取得可能。

Fix: ルート冒頭に認証チェックを追加。

```ts
const supabase = createClient()
const { data: { user }, error } = await supabase.auth.getUser()
if (error || !user) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
}
```

### [SEC-A] ログアウト時の IndexedDB クリア処理（変化なし）

`supabase.auth.signOut()` はコードベース全体に存在しない。スタッフ PIN ログアウト（`localStorage.removeItem`）のみで、Supabase セッションも IndexedDB（trees / pendingEdits 等）もクリアされない。端末引き継ぎ・紛失時のリスクが継続。

Fix: `useStaffPin.ts` の `logout` 関数に以下を追加。

```ts
const logout = useCallback(async () => {
    localStorage.removeItem(STORAGE_KEY)
    const supabase = createClient()
    await supabase.auth.signOut()
    await db.delete()  // IndexedDB 全削除
    setTick(t => t + 1)
}, [])
```
