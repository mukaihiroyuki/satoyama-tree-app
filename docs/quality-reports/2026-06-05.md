### [5] Supabase エラーチェック漏れ（5件）

Supabase クライアントはエラー時に throw しない。`{ data, error }` を返すため `try/catch` では捕捉できない。

| ID | File | Line | Pattern | Impact |
|----|------|------|---------|--------|
| 5-1 | `src/lib/activity-log.ts` | 27 | `.insert()` 結果を非チェック | ログ記録失敗がサイレントに |
| 5-2 | `src/app/c/[clientId]/s/[shipmentId]/page.tsx` | 51 | `{ data: clientData }` のみ分割代入 | DB エラーが「not found」と区別不能 |
| 5-3 | 同上 | 75 | `{ data: shipment }` のみ分割代入 | 同上 |
| 5-4 | 同上 | 101 | `{ data: receipts }` のみ分割代入 | DB エラーが「全件未受入」扱いに |
| 5-5 | `src/hooks/useTrees.ts` | 95 | `Promise.all()` に try/catch なし | rejection が unhandled のまま伝播 |

Fix 例（5-5）: `refreshData` の `Promise.all` を `try { … } catch (err) { console.error('refreshData error:', err) }` で囲む。

### [SEC-1] API ルート認証チェック（1件）

`src/app/api/label/[id]/route.ts` が `.lbx` ファイルを生成して返す。呼び出し元の認証検証なし（`auth.getUser()` なし）。tree ID を知れば誰でも任意ラベルを取得できる可能性がある。

Fix: ルート冒頭で `const { data: { user }, error } = await supabase.auth.getUser()` チェックを追加し、未認証なら 401 を返す。
