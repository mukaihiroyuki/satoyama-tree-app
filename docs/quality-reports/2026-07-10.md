### [SEC-A] ログアウト時の IndexedDB クリア処理（継続）

`src/hooks/useStaffPin.ts:37` の `logout()` は `localStorage.removeItem(STORAGE_KEY)` のみ。Dexie DB（trees / species / pendingEdits / pendingRegistrations）がクリアされない。端末引き継ぎ・紛失時に樹木データ・顧客情報が残留する。

`supabase.auth.signOut()` も未確認（コードベース全体で1件もなし）。Supabase セッションの破棄も行われていない可能性がある。

Fix:
```ts
const logout = useCallback(async () => {
    const { db } = await import('@/lib/db')
    await db.delete()
    // または db.trees.clear(); db.species.clear(); db.pendingEdits.clear(); db.pendingRegistrations.clear()
    localStorage.removeItem(STORAGE_KEY)
    setTick(t => t + 1)
}, [])
```

### [NEW-1] `get_shipped_by_fiscal_year()` RPC の anon グラント（Low）

`supabase/migrations/20260708_shipped_by_fiscal_year_rpc.sql` が `anon` ロールに `EXECUTE` を付与している。関数は `security invoker` なので RLS が適用され実データ漏洩はないが、`authenticated` のみで十分なケースに余分な権限を付与している。

```sql
-- 現状
grant execute on function public.get_shipped_by_fiscal_year() to anon;
-- 推奨: anon グラントを削除
```

実際の呼び出し元（`src/app/page.tsx`）は `createClient()` from `@/lib/supabase/server`（認証済みセッション）を使っており、anon グラントは不要。
