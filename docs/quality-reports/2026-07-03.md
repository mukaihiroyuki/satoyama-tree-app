### [SEC-1] API ルート認証チェック（1件）

`src/app/api/label/[id]/route.ts` に `auth.getUser()` なし。tree ID を知れば未認証でも `.lbx` ファイルを取得できる。

今週のコミット `7f4adee`（ラベルにレイアウト追加）で同ファイルが変更されたが、認証チェックは追加されていない。

Fix: ルート冒頭で以下を追加：
```typescript
const supabase = createClient() // @/lib/supabase/server から
const { data: { user }, error: authError } = await supabase.auth.getUser()
if (authError || !user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
```

### [SEC-A] ログアウト時の IndexedDB クリア処理

`src/hooks/useStaffPin.ts` の `logout()` は `localStorage.removeItem(STORAGE_KEY)` のみ。IndexedDB（Dexie `SatoyamaOfflineDB`）のクリアなし。端末引き継ぎ・紛失時に樹木データ・顧客情報・pendingEdits が残存する。

なお、このアプリのスタッフ認証は Staff PIN（localStorage）であり、`supabase.auth.signOut()` の呼び出し箇所はコードベース全体に存在しない（Supabase Auth セッションはブラウザの Cookie/localStorage に残る）。

Fix（優先度: 中）:
```typescript
const logout = useCallback(async () => {
    const { db } = await import('@/lib/db')
    await db.delete()  // IndexedDB 全削除
    localStorage.removeItem(STORAGE_KEY)
    setTick(t => t + 1)
}, [])
```
