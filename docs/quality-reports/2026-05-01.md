## Recommended Priorities

1. **(High — persists)** `src/lib/activity-log.ts:27` — single insert discards Supabase error. Fix: `const { error } = await supabase.from(...).insert({...}); if (error) console.error(...)`.
2. **(High — new)** `src/app/shipments/[id]/page.tsx:429` — bare `await supabase.from('trees').update(...)` on individual cancel. Tree may silently stay as `shipped` in DB if the update fails.
3. **(High — new)** `src/components/ShipmentDialog.tsx:142` — `await supabase.from('estimates').update(...)` fire-and-forget. Estimate status may silently fail to update on ship.
4. **(Medium — persists)** `src/app/shipments/[id]/page.tsx:252` — `const { count }` missing error; if the count query fails, `count` is `null` and the delete branch is skipped silently.
5. **(Medium)** Fix the 14 `const { data: ... }` patterns in 5a — add `error` destructuring so failures appear in production logs. Highest-impact files: `c/[clientId]/page.tsx` (3 queries) and `c/[clientId]/s/[shipmentId]/page.tsx` (3 queries) since these are customer-facing portal pages.
6. **(Low)** Type the three `any` usages in `tree-repository.ts` (lines 31, 80, 679) using the Supabase-inferred row type.
