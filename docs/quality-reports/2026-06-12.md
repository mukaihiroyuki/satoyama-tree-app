### [SEC-1] API ルート認証チェック（1件、前週から変化なし）

`src/app/api/label/[id]/route.ts` が `.lbx` ファイルを生成して返す。`auth.getUser()` によるセッション検証なし。tree ID を知れば未認証でも任意ラベルを取得できる。

Fix: ルート冒頭に認証チェックを追加。
```ts
const { data: { user }, error: authError } = await supabase.auth.getUser()
if (authError || !user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
```

---

### [SEC-A] ログアウト時の IndexedDB クリア処理（前週から変化なし）

`useStaffPin.ts` の `logout` は `localStorage.removeItem(STORAGE_KEY)` のみ実行し、Dexie の IndexedDB（樹木データ・pendingEdits）をクリアしない。端末紛失・引き継ぎ時にキャッシュデータが残留するリスクあり。

Fix 案: `logout` コールバック内で `db.delete()` を呼び出す（再初期化は次回アクセス時に Dexie が自動実行）。
