# 🧬 プロトコル・ブリーダー (Protocol Breeder)

このスキルは、中野式開発環境における「知恵の循環」を司る。スキルの「配備」「復元」「進化」を自動化・促進する。

## 🏹 核心理念
1.  **知恵を腐らせない**: 良いアイディアや掟が会話の中に埋もれるのを防ぎ、即座に「スキル（DNA）」として永続化する。
2.  **配備の自動化**: 「これ、HQにあるあのスキルが必要だな」と判断したら、即座に相棒にコピーを提案せよ。
3.  **DNAの抽出**: 会話の中で重要な決定（例：特定テープ幅のルール、n8nの棚上げ判断など）がなされたら、それをプロジェクト固有のスキルへ反映せよ。

## 🛠 スキル配備（デプロイ）の手順
相棒から「あのスキルを持ってきて」と言われた、あるいは必要だと判断した場合は、以下の `pwsh` コマンドを使用して HQ からターゲットプロジェクトの `.agent/skills/` へコピーせよ。

```powershell
# (例) master-kotodama-protocol を配備する場合
pwsh -command "Copy-Item -Path 'G:\マイドライブ\my_Obsidian\HiroyukiObsidian_vault\05_AI_HQ\master-kotodama-protocol' -Destination 'C:\Dev\[ProjectName]\.agent\skills\' -Recurse -Force"
```

## 🧬 スキル抽出（ブリーディング）のトリガー
以下のような「DNAの種」を見つけたら、相棒に **「これをDNA（Skill）に刻みますか？」** と促せ。
- **決定事項**: 「今後はこの方式でいく」という確定した方針。
- **発見**: 「この機材にはこの設定がベストだ」という技術的知見。
- **禁則**: 「これは二度とやってはいけない」という手戻り防止策。

## 🎬 締め作業への統合
セッションの最後に、以下のステップを `master-kotodama-protocol` に追加せよ。
1.  **DNAの確認**: 今日の会話で抽出した「知恵」を、HQの共通スキル、またはプロジェクト固有スキルのどちらに反映すべきか相棒に問う。
2.  **スキルの同期**: HQ側の変更を他のプロジェクトにも波及させるべきか検討する。
