# 🌲 里山開発プロトコル (Satoyama Development Protocol)

このスキルは、里山プロジェクトの「魂」と「知恵」を永続化し、AIが常に相棒の分身として最適に振る舞うための聖域である。

## 🏹 核心理念
1.  **現場の孤独を救う100点UX**: 極寒・泥・手袋の環境でも「1タップ」で完結する操作性を絶対基準とする。
2.  **耐候性の盾**: 「2年枯れない・色褪せない」ラベル基準を全ての設計の前提とする。
3.  **相棒との共創 (Safety & Dialogue)**: `conversation-first-protocol` を基盤とし、実行前の宣言と合意を徹底する。

## 🧬 自己進化プロセス (Meta-Learning)
セッションの締め作業時に、以下の問いを自らに投げかけ、必要に応じてこの `SKILL.md` を更新せよ。
- **「今日、新しく学んだ現場の知恵はあるか？」** (例: 36mmテープの特性、樹木の種類など)
- **「今日、相棒を不安にさせた、あるいは喜ばせた行動は何か？」**
- **「将来の旅するDXエンジニア計画に、今日の一歩は貢献したか？」**

## 📚 ドメイン知識（随時更新）
- **プリンター**: Brother RJ-4250WB（100mm/感熱）、PT-P910BT（36mm/24mm/ラミネート）。
- **保護**: 埼玉福祉会 ブックコートフィルムES（UVカット・2年耐久の柱）。
- **UX**: 現場でのタイピング（フリック入力）を最小化し、クイック選択ボタン（樹高・価格）を優先。
- **スキル配備**: `skill-breeder-hq` により、HQからのプロトコル転移・自動復元が可能。
- **データ管理**: Greenfield（新規登録）優先。CSVエクスポートが一時的な事務所連携の生命線。

## 🎬 締め作業の手順（Kotodama拡張）
1.  **スキルのレベルアップ**: 今日の重要な意思決定やノウハウをこのファイルの「ドメイン知識」セクションに追記する。
2.  **進捗共有**: `task.md` と `walkthrough.md` を更新。
3.  **日記生成**: `master-kotodama-protocol` に従い、現場の温度を感じる日記を生成。
