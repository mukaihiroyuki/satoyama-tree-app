# 🛡️ AI Code Editor 安全ガイド (Editor Safety Guide)

このスキルは、AIエージェントがコードを破壊するリスクを最小化し、常に復元可能な状態を維持するための「生存戦略」である。全ての作業において最優先される。

## 🛡️ 絶対鉄則 (Ironclads)

### 1. 完璧な状態の即時固定
- **Git Commit**: 動作が確認できたコードは、説明を添えて即座にコミットする。
- **物理バックアップ**: 特に重要なロジックが完成した際は、ファイルコピーやObsidian等へ全文をバックアップする。

### 2. 壊さないための相談作法
- **差分優先**: 全体を書き換えるのではなく、修正箇所を特定し、差分形式で提示・適用する。
- **ロジック保護**: 重要なロジックの変更前に必ず確認を行う。

### 3. コンテキストの純度維持
- **現状の明示**: 常に最新の動作状況とエラー内容を正確に把握する。

## ⚡ パフォーマンス維持 (Optimization)

AIが固まらず、常にサクサクと応答を返すための「軽量化」を徹底する。

### 1. 「荷降ろし」の徹底
- **不要な添付の解除**: チャット欄に不要なフォルダやファイルが添付されている場合は、即座に解除（×ボタン）する。
- **ピンポイント指定**: 修正が必要なファイルのみを読み込ませるように誘導する。

### 2. 環境の最適化
- **除外設定**: `node_modules`, `.next`, `.git` 等の重いフォルダを、必ず `.vscode/settings.json` の `search.exclude` と `files.watcherExclude` に追加する。
- **プロジェクト初期化**: 新しいプロジェクトを開始する際は、まずこの軽量化設定から着手する。
